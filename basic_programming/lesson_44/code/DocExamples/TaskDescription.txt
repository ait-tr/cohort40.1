Написать приложение для интернет-магазина (backend)

1. Обеспечить получение и обработку запросов
со стороны клиентской части (сторонний frontend или API)
для оформления заказа на покупку товара
(не включает в себя:
функционал склада, т.е. добавление товара, работу с поставщиками;
функционал бухгалтерии: списание товара и прочее)

2. Обеспечить:
 - хранение данных о товарах (список товаров и их количество и цены)
 - хранение информации о заказчиках (идентификация и история покупок)
 - хранение информации о заказах (состав заказа, его характеристики и история)
 - хранение информации о клиентской корзине (недооформленом заказе, т.е. выбранных товарах в корзину, но еще не купленных)
 - регистрация нового пользователя
 - авторизация пользователя для оформления заказа
 - хранение данных о видах оплаты
 - хранение данных о параметрах выполнения заказа (оплачен/неоплачен, собран, отгружен, доставлен)

3. Какие запросы приложение должно обрабатывать:
 Работа с товарами:
 - передать все товары
 - передать список всех категорий
 - передать товары по категории
 - передать товары по категории в ценовом диапазоне
 - передать список всех производителей
 - передать товары по категории и по производителю
 - передать товары по категории и по производителю и по цене
 - передать данные о количестве конкретного товара
 - показать конкретный товар по id

 Работа с пользователем:
 - регистрация
 - авторизация
 - редактирование данных

 Работа с заказами:
 - показать информацию по корзине пользователя
 - внести изменения в корзину
 - оформить заказ (т.е. сформировать заказ и перенести туда товары из корзины, очистить корзину, внести изменения в количество товара на складе в зарезервированные)
 - отмена заказа (по запросу пользователя)
 - передать список всех заказов пользователя
 - передать информацию о конкретном заказе
